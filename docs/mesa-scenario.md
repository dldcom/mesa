## 🎮 핵심 설정

### 기기 환경
- 학생 1인당 태블릿 1대 (외장 키보드 없음)
- 팀 = 4명, 한 반에 다수 팀 병렬 운영
- 입력 방식: **터치 중심** (드래그, 탭, 회전). 짧은 텍스트 입력만 온스크린 키보드 허용

### 실시간 통신
Socket.io 3층 room 구조:
```
session:<code>          ← 한 반 전체 (모든 팀 + 교사)
├ team:<id>             ← 팀 내 4명 상태 동기화
└ teacher:<sessionId>   ← 교사 대시보드 집계 (읽기 전용)
```

### 상태 관리 원칙
- **서버가 상태의 단일 진실 원천(Single Source of Truth)**
- 클라이언트는 서버 상태를 반영만. 클라이언트 간 직접 동기화 금지
- 실시간 상태는 서버 메모리(or Redis), Postgres는 세션 시작/종료 기록용

### 교사 대시보드
- **관찰 전용**. 인게임 개입 기능 없음
- 세션 시작 전 설정, 진행 모니터링(진행률/정체/힌트요청 표시), 게임 후 결과 요약만
- 교사가 필요 시 대면으로 직접 지도

## 🎬 1막 — 전력망 동기화

**교과 개념**: 동분모 분수의 덧셈/뺄셈
**예상 소요**: 7~10분

### 배경 공간
4명의 각 화면 구석에 자신의 시작 노드. 중앙에 텅 빈 메인 배터리. 각 노드에서 중앙으로 향하는 미로 경로에 연산 필터가 배치됨.

### 퍼즐 세팅

각 학생은 **2개의 갈림길** 중 하나를 선택:

| 학생 | 시작 | 경로 ① | 경로 ② |
|---|---|---|---|
| A | 1/8 | +5/8 → 6/8 | **+2/8 → 3/8** ✓ |
| B | 3/8 | **−1/8 → 2/8** ✓ | +1/8 → 4/8 |
| C | 5/8 | −1/8 → 4/8 | **−3/8 → 2/8** ✓ |
| D | 7/8 | **−6/8 → 1/8** ✓ | −3/8 → 4/8 |

**목표**: 네 명의 도착값 합 = 정확히 **1 (= 8/8)**

### 실패 조건
- 합이 1을 초과 → 배터리 과부하 → 리셋
- 합이 1 미만 → 부족 경고 → 재시도

### 정답

**선택 패턴: ② — ① — ② — ①**

| 학생 | 선택 | 도착값 |
|---|---|---|
| A | 경로 ② (+2/8) | 3/8 |
| B | 경로 ① (−1/8) | 2/8 |
| C | 경로 ② (−3/8) | 2/8 |
| D | 경로 ① (−6/8) | 1/8 |
| | **합** | **8/8 = 1** ✓ |

### 유일성 검증
16가지 조합 전부 계산한 결과, 합이 정확히 8/8 되는 조합은 단 하나. 다른 15개 조합은 모두 6/8 ~ 16/8 범위의 다른 값.

### 풀이 과정 (학생들의 추론 경로)
1. 총합이 8/8 이어야 함
2. D의 선택지는 1/8 또는 3/8 (가장 큰 시작값이라 가장 큰 영향)
3. D=3/8 가정 → 남은 셋의 합이 5/8 → 최솟값 3+1+2=6 → **불가능**
4. ∴ D=1/8 확정 → 남은 셋의 합 = 7/8
5. 조합 시도 → (A=3, B=2, C=2) 만 합 7 → 유일해

### 상호작용
- 각자 자기 태블릿에서 두 갈림길 중 하나 탭
- 선택 중에는 다른 학생에게 "내가 +2/8 쪽 갈게" 같은 의논 필요
- 4명 모두 선택 완료 + "확정" 버튼 → 중앙 배터리에 실시간으로 흘러들어감
- 합이 1이 되면 배터리 완충 애니메이션 → 다음 방 열림

## 🎬 2막 — 냉각수 코어 식별

**교과 개념**: 가분수↔대분수 변환, 분수 비교, 논리 추리
**예상 소요**: 10~12분

### 배경 공간
화면 중앙에 거대한 양팔 저울. 하단에 6개의 정육면체 코어가 굴러다님. 코어의 라벨은 긁혀서 무게가 안 보임.

### AI의 경고
> *"위험. **1과 3/4** 크기의 냉각수 코어를 투입구에 넣으십시오."*

### 퍼즐 세팅

6개 코어의 무게는 {**5/4, 6/4, 7/4, 8/4, 9/4, 10/4**} 중 하나씩 (중복 없음).
색깔 6종: 빨강, 파랑, 초록, 노랑, 보라, 주황.

### 단서 배분 (각 학생에게 1장씩)

| 학생 | 단서 |
|---|---|
| A | "**빨강**은 **파랑**보다 **2/4** 더 무겁다." |
| B | "**초록** 2개를 합치면 **노랑** 1개와 같다." |
| C | "**보라**는 **초록**보다 **3/4** 더 무겁다." |
| D | "찾아야 할 코어는 **1과 3/4** (= 7/4)." |

### 보조 도구
- 양팔 저울 (드래그로 코어 올리기, 실시간 수평/기울기 표시)
- 시도 횟수 제한 없음 (단서 추론이 핵심이므로)

### 정답

**파란색 코어 (= 7/4 = 1과 3/4)**

### 풀이 과정

**Step 1 — B의 단서로 초록/노랑 확정**
- `2 × 초록 = 노랑`
- 가능한 쌍: 초록=5/4, 노랑=10/4 (유일, 다른 값은 모두 집합 범위 밖)

**Step 2 — C의 단서로 보라 확정**
- 보라 = 초록 + 3/4 = 5/4 + 3/4 = **8/4**

**Step 3 — A의 단서로 빨강/파랑 확정**
- 남은 {6/4, 7/4, 9/4} 에서 차이 2/4 쌍: **(7/4, 9/4)**
- → 파랑=7/4, 빨강=9/4

**Step 4 — 주황 확정**
- 남은 하나 = **주황 = 6/4**

**Step 5 — 목표 식별 (D의 단서)**
- 1과 3/4 = 가분수로 **7/4** → **파란색**

### 전체 대응표

| 색 | 무게 (가분수) | 대분수 |
|---|---|---|
| 초록 | 5/4 | 1과 1/4 |
| 주황 | 6/4 | 1과 2/4 |
| **파랑** | **7/4** | **1과 3/4** ← 정답 |
| 보라 | 8/4 | 2 |
| 빨강 | 9/4 | 2와 1/4 |
| 노랑 | 10/4 | 2와 2/4 |

### 상호작용
- 각자 자기 단서는 자기 태블릿에만 표시됨 (강제 공유 필요)
- 저울에 코어를 드래그해서 올리면 **4명 태블릿에 동시 반영**
- "파랑 코어 투입" 탭 → 정답이면 안정화, 오답이면 AI가 재시도 허용

## 🎬 3막 — AI 오버라이드 암호

**교과 개념**: 부분과 전체, 시각적 추론
**예상 소요**: 3~5분 (가장 빠른 막)

### 배경 공간
붉은색 메인 서버실. 화면 중앙에 4자리 비밀번호 터치 패드.

### 퍼즐 세팅

**8조각 투명 원판 4장** (각 학생 태블릿에 1장씩). 칸 번호는 12시 방향부터 시계방향 1~8.

| 학생 | 색 | 색칠된 칸 | 색칠 수 |
|---|---|---|---|
| A | 🔴 빨강 | 1, 2, 3, 4, 5 | 5/8 |
| B | 🔵 파랑 | 1, 2, 3, 7 | 4/8 |
| C | 🟢 초록 | 4, 5, 6, 7 | 4/8 |
| D | 🟡 노랑 | 4, 5, 6, 7, 8 | 5/8 |

### 암호 규칙 (AI 안내)

> *자리1 = A와 B의 원판을 겹쳤을 때, **둘 다 색칠된 칸의 개수***
> *자리2 = B와 C의 겹친 칸 수*
> *자리3 = C와 D의 겹친 칸 수*
> *자리4 = A와 D의 겹친 칸 수*

※ **분수 덧셈이 아닌 조각 수 세기 방식**. 중첩 영역의 조각을 직접 카운트.

### 상호작용
- 두 명이 각자 원판을 중앙으로 드래그 → 화면에서 **투명 레이어로 겹쳐져 표시**
- 겹친 원판 위에서 "둘 다 색칠된 칸" 이 시각적으로 강조됨 (예: 진한 보라색)
- 학생들이 해당 칸을 탭하여 카운트

### 정답

**암호: 3 — 1 — 4 — 2**

### 풀이 과정

**자리 1: A ⊕ B**
- A = {1,2,3,4,5}, B = {1,2,3,7}
- 둘 다 색칠: {1, 2, 3} → **3칸**

**자리 2: B ⊕ C**
- B = {1,2,3,7}, C = {4,5,6,7}
- 둘 다 색칠: {7} → **1칸**

**자리 3: C ⊕ D**
- C = {4,5,6,7}, D = {4,5,6,7,8}
- 둘 다 색칠: {4, 5, 6, 7} → **4칸**

**자리 4: A ⊕ D**
- A = {1,2,3,4,5}, D = {4,5,6,7,8}
- 둘 다 색칠: {4, 5} → **2칸**

## 🎬 4막 — 최종 제어실: 에너지 균형

**교과 개념**: 대분수↔가분수, 분수 대소 비교, 동분모 덧셈 (통합)
**예상 소요**: 10~15분 (보스전)

### 배경 공간
거대한 원형 제어실. 천장에 중앙 에너지 게이지. 바닥에 4개의 대형 레버가 **시계방향으로** 배치 (12시, 3시, 6시, 9시).

### AI의 마지막 과제
> *"너희가 진짜 수학을 아는지 증명해라.*
> *내 에너지 밸런스를 너희가 직접 맞춰라.*
> *실패하면 연구소는 폭발한다."*

### 5가지 안정화 조건

> ⚙️ **에너지 밸런스 프로토콜**
>
> ① 네 레버 값의 **합 = 2와 5/8**
> ② 모든 레버는 **진분수** (0 < 값 < 1)
> ③ 모든 분모는 **8**
> ④ 네 레버의 분자는 **서로 다른 자연수**
> ⑤ **시계방향 순서로 값이 점점 커진다**
>    (12시 < 3시 < 6시 < 9시)

### 상호작용
- 각 학생은 자기 위치(12/3/6/9시)의 레버 1개 담당
- 레버 드래그로 분수 값 설정 (분모 8 고정, 분자 자연수)
- 다른 학생의 레버 값은 실시간 작게 표시 (Socket 동기화)
- 4명 동시에 "확정" 버튼 탭 → 5조건 검증
- 위반 시 빨간 경고 + 어떤 조건이 틀렸는지 표시

### 정답

| 위치 (시계방향) | 레버 값 |
|---|---|
| 12시 | **3/8** |
| 3시 | **5/8** |
| 6시 | **6/8** |
| 9시 | **7/8** |
| **합** | **21/8 = 2와 5/8** ✓ |

### 풀이 과정

**Step 1**: 2와 5/8 → 가분수 → **21/8**

**Step 2**: 조건 ②③ → 분자 후보 {1, 2, 3, 4, 5, 6, 7}

**Step 3**: 조건 ④ → 4개 서로 다름
- 4개의 합이 21이 되는 조합 탐색
- 최댓값 {4,5,6,7} = 22 → **1 뺀 값 필요**
- 4를 3으로 교체: {3, 5, 6, 7} = **21** ✓
- 다른 조합 불가 (체계적 검증 완료)

**Step 4**: 조건 ⑤ → 오름차순 배치
- 12시=3/8, 3시=5/8, 6시=6/8, 9시=7/8

### 엔딩 연출
4명 동시 확정 → 검증 통과:
- 중앙 게이지 푸른색 안정화
- 제어실 조명 붉은색 → 흰색 전환
- AI 마지막 메시지:
  > *"확인 완료. 수학을 아는 아이들만이 나를 멈출 수 있다 — 가설 입증.*
  > *너희의 이름을 기록에 남긴다. M.E.S.A 프로젝트, 영구 종료."*
- 연구소 출구 셔터 개방
- 팀별 총 소요 시간 + "너희가 지켜낸 데이터" 엔딩 타이틀

## 🛠 구현을 위한 기술 노트

### Socket 이벤트 명세 (초안)

**학생 → 서버**
```ts
student:join        { sessionCode, teamId, studentName }
puzzle:action       { act, payload }     // 각 막의 조작 (경로 선택, 코어 드래그, 레버 조정 등)
hint:request        { teamId }
```

**서버 → 팀원 (team:<id> 브로드캐스트)**
```ts
team:state          { currentAct, progress, ... }
puzzle:update       { studentId, action }
puzzle:solved       { act, nextAct }
puzzle:failed       { act, reason }
```

**서버 → 교사 (teacher:<sessionId>)**
```ts
team:update         { teamId, progress, stuckTime, lastAction }
team:hintRequest    { teamId }
team:completed      { teamId, totalTime }
```

### 상태 모델 (서버 메모리)

```ts
type SessionState = {
  code: string;
  teams: Map<TeamId, TeamState>;
  startedAt: Date;
}

type TeamState = {
  id: TeamId;
  students: Student[];       // 4명
  currentAct: 1 | 2 | 3 | 4;
  actStates: {
    act1: { selections: (1|2|null)[] };  // 4명의 선택
    act2: { revealedClues: string[], scalePlacements: ... };
    act3: { overlayedDisks: [DiskId, DiskId][], passcodeInput: number[] };
    act4: { leverValues: Fraction[] };   // 4개 레버
  };
  completedActs: number[];
  startedAt: Date;
}
```

### 클라이언트 책임 분담

- **Phaser**: 게임 씬 (배경, 오브젝트, 애니메이션, 드래그 상호작용)
- **React**: HUD, 단서 패널, 메뉴, 확정 버튼, 힌트 요청
- **Socket.io 클라이언트**: 팀 상태 동기화, 서버 이벤트 수신/전송

### 우선순위 구현 순서 제안

1. **프로젝트 골격** + Socket 기본 연결 + 팀 매칭
2. **2막** (기술 난이도 가장 높음 — 저울 실시간 동기화, 단서 분산)
3. **1막** (필터 선택 UI + 합산 애니메이션)
4. **4막** (레버 실시간 동기화 + 5조건 검증)
5. **3막** (원판 투명 오버레이, 가장 마지막)
6. 교사 대시보드
7. 프롤로그/엔딩 시네마틱
